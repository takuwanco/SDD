# Markdown記法の基本

このページでは、仕様駆動開発でよく使用するMarkdown記法をまとめています。

## 見出し

見出しは、`#`の数でレベルを指定します。

| 記法 | 表示例 |
|------|--------|
| `# 見出し1` | 見出し1（最大） |
| `## 見出し2` | 見出し2 |
| `### 見出し3` | 見出し3 |
| `#### 見出し4` | 見出し4 |

**注意**: 見出しのレベルは飛ばさないようにしましょう（`#`の次にいきなり`###`にならないように）。

## 太字・斜体

| 記法 | 表示例 |
|------|--------|
| `**太字**` | **太字** |
| `*斜体*` | *斜体* |
| `***太字斜体***` | ***太字斜体*** |

## リスト

### 箇条書き（順序なし）

```markdown
- 項目1
- 項目2
  - 入れ子の項目1
  - 入れ子の項目2
- 項目3
```

表示例：
- 項目1
- 項目2
  - 入れ子の項目1
  - 入れ子の項目2
- 項目3

### 番号付きリスト（順序あり）

```markdown
1. 項目1
2. 項目2
   1. 入れ子の項目1
   2. 入れ子の項目2
3. 項目3
```

表示例：
1. 項目1
2. 項目2
   1. 入れ子の項目1
   2. 入れ子の項目2
3. 項目3

### チェックボックス（タスクリスト）

```markdown
- [ ] 未完了のタスク
- [x] 完了したタスク
```

表示例：
- [ ] 未完了のタスク
- [x] 完了したタスク

## リンク

| 記法 | 表示例 |
|------|--------|
| `[リンクテキスト](URL)` | [リンクテキスト](URL) |
| `[リンクテキスト](URL "タイトル")` | リンクにマウスを乗せるとタイトルが表示される |
| `<URL>` | URLがそのままリンクになる |

**例**:
```markdown
[GitHub](https://github.com)
[SDDリポジトリ](https://github.com/elvezjp/SDD "仕様駆動開発の練習用リポジトリ")
```

## 表

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
| データ4 | データ5 | データ6 |
```

表示例：

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
| データ4 | データ5 | データ6 |

**注意**: 表の列を揃えるために、パイプ（`|`）の前後にスペースを入れると読みやすくなります。

## コードブロック

### インラインコード

```markdown
`git commit`コマンドを実行します。
```

表示例：`git commit`コマンドを実行します。

### コードブロック（複数行）

言語を指定する場合：

````markdown
```bash
git add .
git commit -m "変更内容の説明"
git push
```
````

言語を指定しない場合：

````markdown
```
コードの内容
```
````

>

表示例：

## コメント（非表示のメモ）

Markdownには公式なコメント記法はありませんが、プレビュー時に表示されないコメントを挿入する方法として、以下の2つが一般的です。

### HTMLコメント形式（推奨）

```markdown
<!-- ここにコメントを記述 -->
<!--
複数行のコメントも可能です
長い説明やメモを残すことができます
-->
```

### リンク形式のコメント

```markdown
[//]: # (ここにコメントを記述)
```

**使い分けの目安**：
- 一般的には**HTMLコメント形式（`<!-- -->`）**を推奨します。互換性が高く、複数行にも対応できます。
- リンク形式は、一部のMarkdownレンダラーで非対応の可能性があるため、使用する環境での対応状況を確認してください。

これらのコメントは、プレビュー時や最終的なドキュメントには表示されませんが、Markdownファイルのソースコードには残ります。チーム内でのメモや、後で確認したい内容を記録する際に便利です。

## 画像

```markdown
![代替テキスト](画像のURL)
![代替テキスト](画像のURL "タイトル")
```

**例**:
```markdown
![仕様駆動開発の全体像](images/spec-driven-dev.png)
```

## エスケープ

特殊文字をそのまま表示したい場合は、バックスラッシュ（`\`）でエスケープします。

```markdown
\*これは斜体にならない\*
\# これは見出しにならない
```

表示例：\*これは斜体にならない\*、\# これは見出しにならない

## 仕様駆動開発でよく使う記法の組み合わせ

### チェックリスト付きのタスク管理

```markdown
## 実装タスク

- [ ] 顧客情報一覧表示APIの実装
- [ ] 顧客情報検索APIの実装
- [x] 顧客情報登録APIの実装（完了）
```

### 仕様の構造化

```markdown
# 機能名

## 概要
機能の概要説明

## 要件
- 要件1
- 要件2

## 制約条件
> この機能は、認証済みユーザーのみが利用可能です。
```

### コード例を含む仕様

```markdown
## API仕様

### エンドポイント
```
GET /api/customers
```

### レスポンス例
```json
{
  "customers": [
    {
      "id": 1,
      "name": "田中太郎"
    }
  ]
}
```
```
