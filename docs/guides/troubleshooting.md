# トラブルシューティング

このリポジトリを使用する際に発生する可能性のある問題と、その対処法をまとめています。

## Cursorでのクローンが失敗する

### エラー: "Repository not found"

**原因**: リポジトリが存在しない、またはアクセス権限がない

**対処法**:
1. リポジトリURLが正しいか確認してください
2. GitHubにログインしているか確認してください
3. リポジトリがパブリックになっているか確認してください

### エラー: "Permission denied"

**原因**: GitHubへの認証ができていない

**対処法**:
1. CursorでGitHubアカウントにサインインしているか確認
2. 「File」>「Preferences」>「Settings」でGitHub認証を確認
3. 必要に応じて、GitHubでCursorへのアクセス許可を確認

## Git操作でエラーが発生する

### エラー: "fatal: not a git repository"

**原因**: Gitリポジトリが初期化されていない

**対処法**:
1. リポジトリを正しくクローンしたか確認
2. 正しいディレクトリにいるか確認（`pwd`コマンドで確認）
3. 必要に応じて、`git init`を実行（ただし、クローンしたリポジトリでは通常不要）

### エラー: "Please tell me who you are"

**原因**: Gitのユーザー情報が設定されていない

**対処法**:
Cursorのターミナルで以下のコマンドを実行：

```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"
```

## ファイルが保存できない

### エラー: "Permission denied" または "Read-only"

**原因**: ファイルの権限の問題

**対処法**:
1. ファイルが読み取り専用になっていないか確認
2. ファイルの所有者を確認（`ls -l`コマンドで確認）
3. 必要に応じて、`chmod`コマンドで権限を変更

## AIチャットが動作しない

### AIが応答しない

**原因**: CursorのAI機能が有効になっていない、またはAPIキーが設定されていない

**対処法**:
1. Cursor 2.0では、Composer-1がデフォルトで使用されるため、通常は設定不要です
2. 無料プランの利用制限に達していないか確認
3. インターネット接続を確認
4. Cursorを再起動してみる

### AIの応答が遅い

**原因**: ネットワークの問題、またはサーバーの負荷

**対処法**:
1. インターネット接続を確認
2. しばらく待ってから再試行
3. Cursorを再起動

## その他の問題

### Cursorが起動しない

**対処法**:
1. システム要件を確認（OSのバージョンなど）
2. Cursorを最新版にアップデート
3. 再インストールを試す

### 日本語が正しく表示されない

**対処法**:
1. Cursorの設定でフォントを確認
2. システムのロケール設定を確認
3. 必要に応じて、フォントを変更

## まだ解決しない場合

以下の方法でサポートを求めることができます：

1. **GitHub Issues**: このリポジトリのIssuesで質問を投稿
2. **本書の該当章**: 第7章の詳細な説明を参照
3. **Cursor公式ドキュメント**: [Cursor公式サイト](https://cursor.com)のドキュメントを参照
