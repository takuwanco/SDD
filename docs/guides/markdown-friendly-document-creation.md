# Markdownにしやすい文書の作り方

このページでは、これから新しくWord、PowerPoint、Excelで文書を作る際に、将来のMarkdown変換を意識した作り方について詳しく解説します。

## はじめに

既存のレガシー文書をMarkdownに変換する方法は、**SDDリポジトリの`docs/conversion/word-excel-conversion-guide.md`**や**`docs/conversion/oasys-ichitaro-conversion-guide.md`**で詳しく説明しています。

しかし、**これから新しく文書を作る際にも、将来のMarkdown変換を意識した作り方**をすることで、変換時の手間を大幅に減らすことができます。このページでは、Word、PowerPoint、Excelそれぞれについて、Markdownに変換しやすい文書の作り方を具体的に説明します。

## Word文書を作る際のポイント

### 1. スタイル機能を必ず使う

**なぜ重要か**
Markdown変換ツール（Pandoc、MarkItDownなど）は、Wordのスタイル機能を使って見出しの階層を認識します。フォントサイズや太字を手動で変更しただけでは、見出しとして認識されません。

**正しい使い方**

1. **見出しスタイルを使う**
   - 「ホーム」タブの「スタイル」から「見出し1」「見出し2」「見出し3」などを選択
   - フォントサイズや太字を手動で変更しない
   - 見出しの階層を意識して、レベルを飛ばさない（見出し1の次にいきなり見出し3にならないように）

2. **本文スタイルを使う**
   - 通常の本文は「標準」スタイルを使う
   - 強調したい部分は「強調」スタイルを使う（太字を手動で適用しない）

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| フォントサイズを大きくして見出しに見せる | 「見出し1」スタイルを使う |
| 太字を手動で適用して強調する | 「強調」スタイルを使う |
| 見出しの階層を飛ばす（見出し1→見出し3） | 見出し1→見出し2→見出し3の順に使う |

### 2. 箇条書き機能を統一する

**なぜ重要か**
Markdown変換ツールは、Wordの箇条書き機能を使ってリスト構造を認識します。「数字」や「・」を手入力しただけでは、リストとして認識されません。

**正しい使い方**

1. **番号付きリスト**
   - 「ホーム」タブの「番号付きリスト」ボタンを使う
   - 手入力で「1.」「2.」などと入力しない

2. **箇条書きリスト**
   - 「ホーム」タブの「箇条書き」ボタンを使う
   - 手入力で「・」や「-」などと入力しない

3. **入れ子のリスト**
   - Tabキーでインデントを調整する
   - 手動でスペースを入力しない

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 手入力で「1. 項目1」と入力する | 「番号付きリスト」ボタンを使う |
| 手入力で「・ 項目1」と入力する | 「箇条書き」ボタンを使う |
| スペースを手動で入力してインデントする | Tabキーでインデントを調整する |

### 3. 表機能を適切に使う

**なぜ重要か**
Markdown変換ツールは、Wordの表機能を使って表構造を認識します。スペースやタブでレイアウトしただけでは、表として認識されません。

**正しい使い方**

1. **表機能を使う**
   - 「挿入」タブの「表」から表を挿入する
   - スペースやタブでレイアウトしない

2. **ヘッダー行を明確にする**
   - 1行目をヘッダー行として、太字や背景色で区別する
   - 「表のスタイル」から「ヘッダー行」を選択する

3. **結合セルを避ける**
   - 可能な限り結合セルを使わない
   - 結合セルが必要な場合は、Markdown変換後に手動で調整する必要があることを認識する

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| スペースやタブで表のように見せる | 「表」機能を使う |
| 結合セルを多用する | 結合セルを避け、シンプルな構造にする |
| ヘッダー行が不明確 | 1行目を太字や背景色で区別する |

### 4. 段落を明確にする

**なぜ重要か**
Markdown変換ツールは、段落の区切りを認識して、Markdownの段落に変換します。改行を多用すると、段落の区切りが不明確になります。

**正しい使い方**

1. **段落区切りを使う**
   - Enterキーで段落を区切る
   - Shift+Enterで改行する（段落内の改行）

2. **改行を多用しない**
   - 見た目を整えるために改行を多用しない
   - 段落の間隔は「段落」ダイアログで調整する

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 見た目を整えるために改行を多用する | 段落の間隔を「段落」ダイアログで調整する |
| 段落の区切りが不明確 | Enterキーで段落を区切る |

## Excelシートを作る際のポイント

### 1. 表の構造を意識する

**なぜ重要か**
Markdown変換ツールは、Excelの表構造を認識して、Markdownの表に変換します。構造が不明確だと、変換時に正しく認識されません。

**正しい使い方**

1. **ヘッダー行を明確にする**
   - 1行目をヘッダー行として、太字や背景色で区別する
   - ヘッダー行には、列の内容を表す明確な名前を付ける

2. **データ行との区別を明確にする**
   - ヘッダー行とデータ行の間に罫線を引く
   - ヘッダー行の背景色を変える

3. **空の行や列を避ける**
   - 表の途中に空の行や列を入れない
   - 表の終わりを明確にする

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| ヘッダー行が不明確 | 1行目を太字や背景色で区別する |
| 表の途中に空の行がある | 空の行を削除する |
| 列名が不明確 | 列名を明確で分かりやすい名前にする |

### 2. 結合セルを避ける

**なぜ重要か**
Markdownの表は、結合セルを直接サポートしていません。結合セルがあると、Markdown変換時に構造が崩れる可能性があります。

**正しい使い方**

1. **結合セルを使わない**
   - 可能な限り結合セルを使わない
   - 同じ値を複数のセルにコピーする

2. **結合セルが必要な場合**
   - Markdown変換後に手動で調整する必要があることを認識する
   - 結合セルの内容を別の列に説明として記載する

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 見た目を整えるために結合セルを多用する | 結合セルを避け、シンプルな構造にする |
| 結合セルの内容が不明確 | 結合セルの内容を別の列に説明として記載する |

### 3. 数式の説明を別セルに記載

**なぜ重要か**
Markdown変換時には、数式の結果（値）は変換されますが、数式そのものは変換されません。数式の意図を理解するために、説明が必要です。

**正しい使い方**

1. **数式の説明を記載する**
   - 複雑な数式を使う場合は、その説明を別セルやコメントに記載する
   - 数式の意図を明確にする

2. **コメント機能を使う**
   - セルにコメントを追加して、数式の説明を記載する
   - 「レビュー」タブの「コメントの挿入」を使う

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 数式の説明がない | 数式の説明を別セルやコメントに記載する |
| 数式の意図が不明確 | 数式の意図を明確にする説明を追加する |

## PowerPointスライドを作る際のポイント

### 1. スライドの構造を明確にする

**なぜ重要か**
Markdown変換ツールは、PowerPointのスライド構造を認識して、Markdownの見出しや段落に変換します。構造が不明確だと、変換時に正しく認識されません。

**正しい使い方**

1. **スライドの種類を明確にする**
   - タイトルスライド：プレゼンテーションのタイトル
   - 見出しスライド：セクションの見出し
   - 本文スライド：内容を説明するスライド

2. **スライドマスターを使う**
   - 「表示」タブの「スライドマスター」で、スライドの種類を定義する
   - スライドの種類に応じて、レイアウトを統一する

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| スライドの種類が不明確 | スライドマスターで種類を定義する |
| レイアウトが統一されていない | スライドマスターでレイアウトを統一する |

### 2. 箇条書きを統一する

**なぜ重要か**
Wordと同様に、PowerPointでも箇条書き機能を使うことで、Markdown変換時にリスト構造が正しく認識されます。

**正しい使い方**

1. **箇条書き機能を使う**
   - 「ホーム」タブの「箇条書き」ボタンを使う
   - 手入力で「・」や「数字」などと入力しない

2. **入れ子のリスト**
   - Tabキーでインデントを調整する
   - 手動でスペースを入力しない

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 手入力で「・ 項目1」と入力する | 「箇条書き」ボタンを使う |
| スペースを手動で入力してインデントする | Tabキーでインデントを調整する |

### 3. 表や図は別途管理

**なぜ重要か**
複雑な表や図は、Markdown変換時に構造が崩れる可能性があります。別ファイルとして管理することで、構造が保たれやすくなります。

**正しい使い方**

1. **表は別ファイルとして管理**
   - 複雑な表は、Excelファイルとして別途管理する
   - スライドには、表の参照（ファイル名やパス）を記載する

2. **図は別ファイルとして管理**
   - 図は、画像ファイル（PNG、SVGなど）として別途管理する
   - スライドには、図の参照（ファイル名やパス）を記載する

**よくある間違いと対処法**

| 間違い | 正しい方法 |
|--------|-----------|
| 複雑な表をスライドに直接埋め込む | 表をExcelファイルとして別途管理する |
| 図をスライドに直接埋め込む | 図を画像ファイルとして別途管理する |

## 実践的なチェックリスト

文書を作成した後、以下のチェックリストで確認してください。

### Word文書のチェックリスト

- [ ] すべての見出しが「見出し1」「見出し2」などのスタイルを使っているか
- [ ] 見出しの階層が飛んでいないか（見出し1→見出し2→見出し3の順になっているか）
- [ ] 箇条書きが「箇条書き」ボタンや「番号付きリスト」ボタンで作成されているか
- [ ] 表が「表」機能で作成されているか（スペースやタブでレイアウトしていないか）
- [ ] 段落の区切りが明確か（改行を多用していないか）

### Excelシートのチェックリスト

- [ ] 1行目がヘッダー行として明確に区別されているか（太字や背景色で）
- [ ] 表の途中に空の行や列がないか
- [ ] 結合セルを避けているか（または、結合セルの内容を別の列に説明として記載しているか）
- [ ] 複雑な数式の説明が別セルやコメントに記載されているか

### PowerPointスライドのチェックリスト

- [ ] スライドの種類（タイトル、見出し、本文）が明確か
- [ ] 箇条書きが「箇条書き」ボタンで作成されているか
- [ ] 複雑な表や図が別ファイルとして管理されているか

## まとめ

これらのポイントを意識することで、将来Markdownに変換する際に、**「構造の回復」が容易**になります。組織の標準がWord、PowerPoint、Excelであっても、これらのポイントを守ることで、Doc-as-Codeの思想に沿った文書管理が可能になります。

**重要なのは、「形式そのもの」ではなく「その形式をどう使うか」**です。Markdown変換を意識した文書作成を心がけることで、仕様駆動開発の流れの中に文書を組み込みやすくなります。
