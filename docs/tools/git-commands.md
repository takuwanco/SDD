# Gitコマンド一覧

このページでは、仕様駆動開発の練習でよく使用するGitコマンドをまとめています。

## gitコマンドとCursorのボタン対応表

下記の表は、よく使うGitコマンドごとに「コマンドの説明」「Cursorでの操作方法」「チャットパネルでの指示例」をまとめたものです。一部の操作（ブランチ管理やコミット履歴の確認など）はターミナルが必要な場合があります。

| Gitコマンド | 説明 | Cursorでの操作方法 | チャットパネルでの指示 |
|------------|------|-------------------|---------------------|
| `git status` | 変更されたファイルの一覧を確認 | Source Controlパネル（⌘+Shift+G）を開く | 「git statusを実行して」または「変更されたファイルの状態を確認して」 |
| `git add .` | すべての変更をステージング | Source Controlパネルの「Changes」セクションの上にある「+」ボタン | 「すべての変更をステージングして」 |
| `git add ファイル名` | 特定のファイルをステージング | ファイルの横にある「+」ボタン | 「README.mdをステージングして」 |
| `git commit -m "メッセージ"` | コミットを実行 | コミットメッセージを入力して「✓」ボタン（⌘+Enter） | 「コミットして」または「変更をコミットして」 |
| `git push` | リモートリポジトリにプッシュ | 「...」メニューから「Push」を選択、または「Sync Changes」ボタン（上向き矢印） | 「プッシュして」 |
| `git pull` | リモートリポジトリから最新の変更を取得 | 「...」メニューから「Pull」を選択、またはステータスバーの同期アイコン | 「最新の変更を取得して」または「プルして」 |
| `git switch ブランチ名` | ブランチを切り替え（旧: `git checkout`） | ステータスバーのブランチ名をクリックして切り替え | 「ブランチ名に切り替えて（例：mainブランチに切り替えて）」 |
| `git switch -c ブランチ名` | 新しいブランチを作成して切り替え（旧: `git checkout -b`） | ステータスバーのブランチ名をクリック→「Create new branch...」 | 「ブランチ名を作成して切り替えて（例：featureブランチを作成して切り替えて）」 |
| `git diff` | 変更内容をdiff形式で表示 | 変更されたファイルをクリック | 「変更の差分を表示して」または「git diffを実行して」 |
| `git reset ファイル名` | ステージングを取り消し | 「Staged Changes」セクションのファイルの横の「-」ボタン | 「README.mdのステージングを取り消して」 |
| `git restore ファイル名` | ファイルの変更を取り消し（**⚠️ 注意：変更が失われます**） | ファイルを右クリック→「Discard Changes」 | 「README.mdの変更を取り消して」 |
| `git branch` | ブランチの一覧を確認 | **ターミナルを利用する必要あり** | 「ブランチの一覧を表示して」または「git branchを実行して」 |
| `git branch ブランチ名` | 新しいブランチを作成 | **ターミナルを利用する必要あり** | 「ブランチ名を作成して（例：featureブランチを作成して）」 |
| `git branch -d ブランチ名` | ブランチを削除 | **ターミナルを利用する必要あり** | 「ブランチ名を削除して（例：featureブランチを削除して）」 |
| `git branch -D ブランチ名` | ブランチを強制削除（**⚠️ 危険：マージされていない変更も削除されます**） | **ターミナルを利用する必要あり** | 「ブランチ名を強制削除して（例：featureブランチを強制削除して）」 |
| `git log` | コミット履歴を確認 | **ターミナルを利用する必要あり** | 「コミット履歴を表示して」または「git logを実行して」 |
| `git log --oneline` | コミット履歴を簡潔に表示 | **ターミナルを利用する必要あり** | 「コミット履歴を簡潔に表示して」 |
| `git log --graph --oneline --all` | コミット履歴をグラフ形式で表示 | **ターミナルを利用する必要あり** | 「コミット履歴をグラフ形式で表示して」 |
| `git show コミットハッシュ` | 特定のコミットの内容を確認 | **ターミナルを利用する必要あり** | 「最新のコミットの内容を表示して」または「このコミットの内容を表示して」 |
| `git commit --amend` | 最後のコミットを修正（**⚠️ 注意：既にプッシュした場合は危険**） | **ターミナルを利用する必要あり** | 「最後のコミットを修正して」または「最後のコミットメッセージを修正して」 |
| `git merge ブランチ名` | ブランチをマージ | **ターミナルを利用する必要あり** | 「ブランチ名をマージして（例：featureブランチをマージして）」 |
| `git rebase ブランチ名` | ブランチをリベース（**⚠️ 注意：既にプッシュしたコミットに対しては危険**） | **ターミナルを利用する必要あり** | 「ブランチ名をリベースして（例：featureブランチをリベースして）」 |
| `git reset --soft HEAD~1` | 最後のコミットを取り消す（変更は保持）（**⚠️ 注意：既にプッシュした場合は危険**） | **ターミナルを利用する必要あり** | 「最後のコミットを取り消して（変更は保持）」 |
| `git remote -v` | リモートリポジトリの情報を確認 | **ターミナルを利用する必要あり** | 「リモートリポジトリの情報を表示して」または「git remote -vを実行して」 |
| `git remote add origin URL` | リモートリポジトリを追加 | **ターミナルを利用する必要あり** | 「リモートリポジトリを追加して（URLは指定が必要）」 |
| `git fetch` | リモートリポジトリの情報を取得（マージはしない） | **ターミナルを利用する必要あり** | 「リモートリポジトリの情報を取得して」または「git fetchを実行して」 |

**注意**: 上記の表には、CursorのGUIで実行できる操作と、ターミナルを利用する必要がある操作の両方を含めています。ターミナルを利用する操作については、Cursorの統合ターミナル（⌘+`）を使用して実行できます。

## 基本的な操作（詳細説明）

### リポジトリの状態を確認する

```bash
git status
```

現在のブランチ、変更されたファイル、ステージングエリア（コミット対象の変更を一時的に保存する場所）の状態を確認できます。

**Cursorでの操作方法**: 左側のサイドバーから「Source Control」アイコン（⌘+Shift+G）をクリックすると、変更されたファイルの一覧が表示されます。ステータスバー（画面下部）に現在のブランチ名も表示されます。

**チャットパネルでの指示**: 「git statusを実行して」または「変更されたファイルの状態を確認して」

### 変更をステージングエリア（コミット対象に追加）に追加する

```bash
```bash
# Tip: コミットするべきでないファイルが含まれていないか、事前に git status で確認しましょう
git status

# すべての変更を追加
git add .

# 特定のファイルを追加
git add README.md

# 特定のディレクトリを追加
git add docs/
```

**Cursorでの操作方法**: Source Controlパネルで、変更されたファイルの横にある「+」ボタンをクリックすると、そのファイルがステージングされます。「Changes」セクションの上にある「+」ボタンをクリックすると、すべての変更を一度にステージングできます。

**チャットパネルでの指示**: 
- すべての変更をステージングする場合：「すべての変更をステージングして」
- 特定のファイルをステージングする場合：「README.mdをステージングして」

### コミットする

```bash
git commit -m "コミットメッセージ"
```

変更をローカルリポジトリに記録します。コミットメッセージは、何を変更したかが分かるように簡潔に記述してください。

**Cursorでの操作方法**: Source Controlパネルの上部にあるテキストボックスにコミットメッセージを入力し、「✓」ボタン（または⌘+Enter）をクリックするとコミットできます。ステージングされていない変更がある場合は、自動的にステージングしてからコミットするかどうかを選択できます。

**チャットパネルでの指示**: 「コミットして」または「変更をコミットして」

### リモートリポジトリにプッシュする

```bash
git push
```

ローカルリポジトリの変更をリモートリポジトリに送信します。

**Cursorでの操作方法**: Source Controlパネルの「...」メニュー（右上）から「Push」を選択します。また、コミット後にSource Controlパネルに表示される「Sync Changes」ボタン（上向き矢印のアイコン）をクリックしてもプッシュできます。

**チャットパネルでの指示**: 「プッシュして」

### リモートリポジトリから最新の変更を取得する

```bash
git pull
```

リモートリポジトリの最新の変更をローカルリポジトリに取り込みます。

**Cursorでの操作方法**: Source Controlパネルの「...」メニューから「Pull」を選択します。または、ステータスバーに表示される同期アイコンをクリックすることもできます。

**チャットパネルでの指示**: 「最新の変更を取得して」または「プルして」

## リポジトリの操作

### リモートリポジトリをクローンする

```bash
git clone https://github.com/ユーザー名/リポジトリ名.git
```

リモートリポジトリをローカル環境にコピーします。

### GitHubリポジトリをアーカイブする

**注意**: この操作はGitコマンドではなく、GitHubのWebインターフェースでの操作です。

GitHubでは、リポジトリをアーカイブすることができます。アーカイブされたリポジトリは読み取り専用になり、新しいコミットやIssue、Pull Requestの作成はできなくなりますが、既存の内容はすべて保持され、いつでも参照可能です。

**GitHubでの操作方法**:
1. リポジトリの「Settings」タブを開く
2. ページ下部の「Danger Zone」セクションを開く
3. 「Archive this repository」をクリック
4. 確認ダイアログでリポジトリ名を入力してアーカイブを実行

**用途**: 活用頻度が低いレガシー文書を管理するリポジトリを「参照用」として保管する際に有効です。

**注意**: アーカイブされたリポジトリは、必要に応じてアーカイブを解除することもできます。

### リモートリポジトリの情報を確認する

```bash
git remote -v
```

設定されているリモートリポジトリのURLを確認できます。

**チャットパネルでの指示**: 「リモートリポジトリの情報を表示して」または「git remote -vを実行して」

### リモートリポジトリを追加する

```bash
git remote add origin https://github.com/ユーザー名/リポジトリ名.git
```

新しいリモートリポジトリを追加します。

**チャットパネルでの指示**: 「リモートリポジトリを追加して（URLは指定が必要）」

## ブランチの操作

### ブランチの一覧を確認する

```bash
# ローカルブランチの一覧
git branch

# リモートブランチも含めた一覧
git branch -a
```

**チャットパネルでの指示**: 「ブランチの一覧を表示して」または「git branchを実行して」

### 新しいブランチを作成する

```bash
git branch ブランチ名
```

新しいブランチを作成します（作成後は現在のブランチのままです）。

**チャットパネルでの指示**: 「ブランチ名を作成して（例：featureブランチを作成して）」

### ブランチを切り替える

```bash
# 推奨
git switch ブランチ名

# 旧コマンド
git checkout ブランチ名
```

**Cursorでの操作方法**: ステータスバー（画面下部）のブランチ名をクリックすると、ブランチの切り替えや作成ができます。または、Source Controlパネルの「...」メニューから「Branch」>「Checkout to...」を選択することもできます。

**チャットパネルでの指示**: 「ブランチ名に切り替えて（例：mainブランチに切り替えて）」

### ブランチを作成して切り替える（一度に）

```bash
# 推奨
git switch -c ブランチ名

# 旧コマンド
git checkout -b ブランチ名
```

**Cursorでの操作方法**: ステータスバーのブランチ名をクリックし、「Create new branch...」を選択して、新しいブランチ名を入力します。

**チャットパネルでの指示**: 「ブランチ名を作成して切り替えて（例：featureブランチを作成して切り替えて）」

### ブランチを削除する

```bash
# ローカルブランチを削除
git branch -d ブランチ名

# 強制削除（マージされていないブランチも削除）
git branch -D ブランチ名
```

**⚠️ 注意**: `git branch -D`は強制削除コマンドです。マージされていない変更も含めてブランチを削除するため、**データが失われる可能性があります**。通常は`git branch -d`を使用し、マージ済みのブランチのみ削除することを推奨します。

**チャットパネルでの指示**: 「ブランチ名を削除して（例：featureブランチを削除して）」。強制削除が必要な場合のみ「ブランチ名を強制削除して（例：featureブランチを強制削除して）」

### ブランチをマージする

```bash
git merge ブランチ名
```

指定したブランチを現在のブランチにマージします。

**チャットパネルでの指示**: 「ブランチ名をマージして（例：featureブランチをマージして）」

### ブランチをリベースする

```bash
git rebase ブランチ名
```

指定したブランチを現在のブランチにリベースします。

**⚠️ 注意**: リベースはコミット履歴を書き換える操作です。**既にプッシュしたコミットに対してリベースを実行すると、他の人と履歴がずれて問題が発生する可能性があります**。プッシュ前のローカルブランチでのみ使用することを推奨します。既にプッシュしたブランチをリベースした場合は、`git push --force`が必要になりますが、共有リポジトリでは避けてください。

**チャットパネルでの指示**: 「ブランチ名をリベースして（例：featureブランチをリベースして）」

### リモートリポジトリの情報を取得する（マージはしない）

```bash
git fetch
```

リモートリポジトリの最新の情報を取得しますが、ローカルブランチにはマージしません。

**チャットパネルでの指示**: 「リモートリポジトリの情報を取得して」または「git fetchを実行して」

## 変更の確認と比較

### 変更内容を確認する

```bash
# ステージング（コミット対象に追加）されていない変更を確認
git diff

# ステージング（コミット対象に追加）された変更を確認
git diff --staged

# 特定のファイルの変更を確認
git diff README.md
```

**Cursorでの操作方法**: Source Controlパネルで変更されたファイルをクリックすると、変更内容がdiff形式で表示されます。行番号の横に「+」や「-」のマークが表示され、追加・削除された行が色分けされて表示されます。

**チャットパネルでの指示**: 「変更の差分を表示して」または「git diffを実行して」

### コミット履歴を確認する

```bash
# 簡潔な履歴
git log --oneline

# 詳細な履歴
git log

# グラフ形式で表示
git log --graph --oneline --all
```

**チャットパネルでの指示**: 「コミット履歴を表示して」または「git logを実行して」。簡潔に表示する場合は「コミット履歴を簡潔に表示して」。グラフ形式の場合は「コミット履歴をグラフ形式で表示して」

### 特定のコミットの内容を確認する

```bash
git show コミットハッシュ
```

**チャットパネルでの指示**: 「最新のコミットの内容を表示して」または「このコミットの内容を表示して」

## 変更の取り消し

### ステージング（コミット対象から除外）を取り消す

```bash
# すべてのステージングを取り消す
git reset

# 特定のファイルのステージングを取り消す
git reset README.md
```

**Cursorでの操作方法**: Source Controlパネルの「Staged Changes」セクションで、ファイルの横にある「-」ボタンをクリックすると、そのファイルのステージング（コミット対象に追加）が取り消されます。

**チャットパネルでの指示**: 「README.mdのステージングを取り消して」

### ファイルの変更を取り消す（注意：変更が失われます）

```bash
git restore ファイル名
```

**チャットパネルでの指示**: 「README.mdの変更を取り消して」

**Cursorでの操作方法**: Source Controlパネルで変更されたファイルを右クリックし、「Discard Changes」を選択します。確認ダイアログが表示されるので、承認すると変更が取り消されます。

### 最後のコミットを取り消す（コミットメッセージを変更）

```bash
git commit --amend -m "新しいコミットメッセージ"
```

**⚠️ 注意**: `git commit --amend`はコミット履歴を書き換える操作です。**既にプッシュしたコミットを修正する場合は、`git push --force`が必要になりますが、他の人と共有しているリポジトリでは避けてください**。共有リポジトリで実行すると、他の人の作業に影響を与える可能性があります。

**チャットパネルでの指示**: 「最後のコミットを修正して」または「最後のコミットメッセージを修正して」

### 最後のコミットを取り消す（変更は保持）

```bash
git reset --soft HEAD~1
```

最後のコミットを取り消しますが、変更内容はステージングエリア（コミット対象の変更を一時的に保存する場所）に保持されます。

**⚠️ 注意**: `git reset`はコミット履歴を書き換える操作です。**既にプッシュしたコミットを取り消す場合は、`git push --force`が必要になりますが、他の人と共有しているリポジトリでは避けてください**。共有リポジトリで実行すると、他の人の作業に影響を与える可能性があります。

**チャットパネルでの指示**: 「最後のコミットを取り消して（変更は保持）」

## よく使うコマンドの組み合わせ

### 変更を確認してからコミット・プッシュ

```bash
# 1. 状態を確認
git status

# 2. 変更を確認
git diff

# 3. 変更を追加
git add .

# 4. コミット
git commit -m "変更内容の説明"

# 5. プッシュ
git push
```

### リモートリポジトリの最新状態を取得してから作業

```bash
# 1. 最新の変更を取得
git pull

# 2. 作業を行う
# （ファイルを編集）

# 3. 変更を確認
git status
git diff

# 4. コミット・プッシュ
git add .
git commit -m "変更内容の説明"
git push
```

## CursorでのGit操作（まとめ）

Cursorでは、Source ControlパネルからGUIでGit操作を行うことができます。多くの操作はコマンドラインを使わずに実行できます：

### 基本的な操作フロー

1. **左側のサイドバー**から「Source Control」アイコン（⌘+Shift+G）をクリック
2. 変更されたファイルが「Changes」セクションに表示されます
3. ファイルの横の「+」ボタンでステージング（コミット対象に追加）（または「Changes」セクションの上の「+」で一括ステージング）
4. 上部のテキストボックスにコミットメッセージを入力して「✓」ボタン（⌘+Enter）でコミット
5. 「...」メニューから「Push」を選択してプッシュ、またはコミット後に表示される「Sync Changes」ボタン（上向き矢印のアイコン）をクリック

### その他の便利な機能

- **変更の確認**: ファイルをクリックすると、diffが表示されます
- **ブランチの切り替え**: ステータスバーのブランチ名をクリック
- **プル**: 「...」メニューから「Pull」を選択
- **変更の取り消し**: ファイルを右クリックして「Discard Changes」を選択

## トラブルシューティング

### コンフリクトが発生した場合

```bash
# 1. コンフリクトの状態を確認（"both modified" と表示されるファイルがある）
git status

# 2. ファイルを開き、コンフリクトマーカー（<<<<<<<, =======, >>>>>>>）を探して修正する
# 例：
# <<<<<<< HEAD
# 現在の変更
# =======
# 相手の変更
# >>>>>>> feature/xxx
#
# ↑これらを削除し、正しいコードだけを残す

# 3. 修正したらステージング
git add .

# 4. マージを完了
git commit -m "コンフリクトを解決"
```

### 間違えてコミットした場合

```bash
# 最後のコミットを取り消す（変更は保持）
git reset --soft HEAD~1

# 変更を確認してから再度コミット
git status
git commit -m "正しいコミットメッセージ"
```

### リモートリポジトリと同期が取れていない場合

```bash
# リモートの状態を確認
git fetch

# リモートとローカルの差分を確認
git log HEAD..origin/main

# 必要に応じてプル
git pull
```
