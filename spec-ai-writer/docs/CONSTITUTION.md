# 原則決定工程

このファイルは、仕様駆動開発の**原則決定工程**で作成されるプロジェクト憲章（Constitution）です。

## プロジェクトの存在意義（Why）

### 背景

仕様駆動開発（SDD）では7つの工程で仕様書を作成するが、各工程で何をどこまで書けばよいか分からず、仕様書の品質がばらつく問題がある。
特に以下の課題が顕在化している：

- 仕様書作成の属人化：経験者でないと何を書くべきか判断できない
- 作成コストの高さ：7工程分の仕様書を手作業で書くには時間がかかりすぎる
- SDDの導入障壁：方法論は優れていても、実践のハードルが高い

### 目的

- AIとの対話形式のインタビューにより、SDD仕様書の作成を自動化する
- SDD方法論の実践ハードルを下げ、誰でも一定品質の仕様書を作成できるようにする
- 7つの工程を順番にガイドし、必要な情報を漏れなく収集する

## 基本原則（Principles）

このプロジェクトでは、以下の原則を優先します：

1. **対話による情報収集**：ユーザーに一方的にフォームを埋めさせるのではなく、AIとの自然な対話で必要な情報を引き出す
2. **SDD方法論への準拠**：7つの工程の定義・必須項目・依存関係を忠実に反映する
3. **段階的な成果物生成**：各工程の完了時にMarkdown仕様書を即座に出力し、進捗を実感できるようにする
4. **中断・再開の保証**：インタビューの状態を永続化し、いつでも中断・再開できるようにする

## スコープと責任範囲（Scope）

### 含まれるもの（In Scope）

- 7工程に対応したAIインタビュー機能
- 工程ごとのMarkdown仕様書の自動生成（Jinja2テンプレート）
- CLIインターフェース（ターミナルでの対話）
- WebUIインターフェース（ブラウザでのチャット形式）
- 複数LLMプロバイダー対応（Claude API / OpenAI API / OpenRouter / ローカル LLM（Ollama・LM Studio・llama.cpp 等の OpenAI 互換エンドポイント）/ AWS Bedrock）
- Web UI（ダッシュボード）からの LLM 設定編集（プロバイダ・Base URL・モデル・API キーをサーバー再起動なしで切替）
- インタビュー状態の永続化と中断・再開
- 生成した仕様書のGit自動コミット

### 含まれないもの（Out of Scope）

- 生成済み仕様書のエディタ機能（仕様書の編集はユーザーが直接MDファイルを編集する）
- 複数人での同時編集・リアルタイムコラボレーション
- 外部プロジェクト管理ツールとの連携（Jira、Backlogなど）
- 仕様書からのコード自動生成

## 制約条件（Constraints）

### 技術的制約

- バックエンド：Python / FastAPI
- フロントエンド：React / TypeScript / Vite
- LLM API呼び出しのレイテンシ（1質問あたり数秒〜十数秒）
- LLM APIの利用コスト（トークン課金）

### 運用上の制約

- LLM APIキーの管理はユーザー側の責任（.envファイルで設定）
- インターネット接続が必須（LLM API呼び出しのため）

### 品質に関する制約

- LLMの回答品質はモデルに依存する（Haiku < Sonnet < Opus）
- 構造化データの抽出精度がモデル性能に左右される

## ガバナンス（Governance）

### 意思決定プロセス

- **SDD方法論に関する判断**：書籍「仕様駆動開発」の定義に準拠する
- **技術選定・アーキテクチャ変更**：開発チームで合議の上、決定する
- **LLMプロンプトの改善**：テスト結果を基に継続的に改善する

### 承認フロー

1. 変更提案（Issue / PR）
2. コードレビュー
3. テスト通過の確認
4. マージ・リリース

## ステークホルダー

- **SDDを実践するプロジェクトマネージャー・開発リーダー**：主要ユーザー
- **SDD初学者**：方法論を学びながら仕様書を作成したいユーザー
- **開発チーム**：本ツールの実装・保守を担当

## 成功基準（Success Criteria）

このプロジェクトが成功したと判断される基準：

1. ユーザーがAIとの対話のみで7工程の仕様書を完成できる
2. 各工程の完了判定が正しく機能し、必要な情報が揃った時点で次の工程に自動的に進む
3. 生成された仕様書がSDDのテンプレート構造に準拠している
4. インタビューを中断・再開しても、コンテキストが正しく引き継がれる
