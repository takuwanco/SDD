# ドキュメントと実装の乖離修正 (Issue #62)

このファイルは、仕様駆動開発の**7つの工程**のうち原則決定工程を除く6工程を一つのドキュメントにまとめた記録です。
今回の変更は軽微なドキュメント修正であるため、全工程を1ファイルで管理します。

---

## ② 企画・要件定義工程

### 目的

`spec-ai-writer/` の実装が進むにつれ、各ドキュメントの記述が実際の実装と乖離するようになりました。
この乖離を放置すると、ユーザーがセットアップ時に詰まったり、誤った手順を実行する原因となります。
本修正では、ドキュメントを実装の現状に合わせ、信頼できる情報源としての品質を回復します。

### 背景

`spec-ai-writer/` に Python・TypeScript の実行可能コードが追加・更新されていく中で、以下の問題が蓄積していました。

- **Python バージョン要件の誤記**: ドキュメント上は 3.9 以上と記載されているが、実際には 3.10 以上が必要
- **存在しない CLI コマンドの記載**: `spec init` コマンドがドキュメントに載っているが実装されていない
- **LLM モデル ID の誤記**: Claude Sonnet 4.6 のモデル ID に誤った日付サフィックスが付いている
- **未使用の環境変数**: `.env.example` に定義されているが `settings.py` に対応フィールドのない変数が残っている
- **リポジトリ構成の説明が実態と乖離**: `SECURITY.md` や `CONTRIBUTING.md` が「Markdown ファイルのみ」という古い前提で書かれたまま

### 修正スコープ

| 優先度 | 問題の種類 | 影響範囲 |
|-------|----------|---------|
| 高 | Python バージョン要件の誤記 | セットアップ手順を読んだユーザー全員 |
| 高 | 存在しない `spec init` コマンドの記載 | CLI 初回利用者 |
| 中 | Claude Sonnet 4.6 のモデル ID 誤記 | LLM 設定をカスタマイズするユーザー |
| 中 | `.env.example` の未使用変数 | 環境構築時に変数の意味を調べるユーザー |
| 中 | README に `APP_ENV` の説明がない | 本番環境を構築するユーザー |
| 低 | `SECURITY.md` / `CONTRIBUTING.md` の構成説明が実態と乖離 | コントリビューター・セキュリティ担当 |

- 今回の変更はすべてドキュメント修正のみ（コード変更なし）
